# Service Calendar & Dispatch Web Application\n\nA full-stack system for managing service jobs, technician dispatch, and reporting.\n\n## 🚀 Tech Stack\n\n- **Frontend:** Next.js (React), Tailwind CSS\n- **Backend & Database:** Supabase (PostgreSQL, Authentication, Storage)\n- **External APIs:** Google Maps Distance Matrix, Telegram Bot API\n- **Deployment:** Vercel (frontend), Supabase (backend + Edge Functions)\n\n## 📂 Project Structure\n\n```\nservice-calendar/\n├── app/\n│   ├── api/              # Backend API routes\n│   ├── auth/             # Authentication pages\n│   ├── admin/            # Admin dashboard\n│   ├── technician/       # Technician dashboard\n│   └── layout.tsx        # Root layout\n├── components/           # Reusable components\n├── lib/                  # Utilities and types\n├── supabase/             # Database migrations\n└── public/               # Static assets\n```\n\n## 🔧 Setup\n\n### 1. Install Dependencies\n\n```bash\nnpm install\n```\n\n### 2. Create Supabase Project\n\n1. Go to [Supabase](https://supabase.com)\n2. Create a new project\n3. Run the SQL migration from `supabase/migrations/001_init_schema.sql`\n\n### 3. Configure Environment Variables\n\n```bash\ncp .env.example .env.local\n```\n\nFill in your Supabase credentials:\n\n```env\nNEXT_PUBLIC_SUPABASE_URL=your_url\nNEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key\nSUPABASE_SERVICE_ROLE_KEY=your_service_role_key\nNEXT_PUBLIC_GOOGLE_MAPS_API_KEY=your_google_maps_key\n```\n\n### 4. Run the Application\n\n```bash\nnpm run dev\n```\n\nVisit http://localhost:3000\n\n## 👥 User Roles\n\n- **Admin**: Create jobs, dispatch technicians, approve continuations\n- **Technician**: Accept jobs, check-in/out, upload work proof\n\n## 📋 Features by Phase\n\n- **Phase 1:** Database Schema & RLS ✅\n- **Phase 2:** Authentication UI & Dashboards ✅\n- **Phase 3:** Technician Mobile View (GPS & Work Proof) ⏳\n- **Phase 4-10:** Advanced Features 📅\n\n## 📝 License\n\nMIT\n
